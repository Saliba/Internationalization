Internationalization - i18n

Why ?

When you are building a web application you will problably want to target as many users as you can, to do so, you must to be available globaly, but when it comes to the point of receiving foreing visitors you should make your website more friendly to them.

How ?

You have a lot of options to translate your website, but today I will talk about angular-gettext, the translator that we use here in Packdocs. By the way, AngularJs has support for currency and number locale formating.

When ?

You should start translating your website since the creation of it, but most of the time, on the begging you don't have time/hands to do it. The good new is that you can start doing it whenever you want.

Can I easily translate an existing app ?

Yes, using angular-gettext you can translate some parts of the site without any impact, it can also help you to find the texts that you haven't translated yet based on a configuration parameter.

Another option is use angular-gettext to extract all the words that are configured to be translated.

You can use a tool called Poedit to translate each sentence.

You can change the current language on the header menu without reloading the page.
